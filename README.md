# AA2 — Android Auto head unit (Python)

Код и ресурсы — в каталоге **`source/`**. В репозитории также этот **`README.md`**.

## Подготовка

```bash
cd source
python3 -m venv venv
./venv/bin/pip install -r requirements.txt
```

Нужны **Pygame**, **PyAV** и **NumPy** (кадр из декодера → `to_ndarray` → pygame). Без NumPy видео не декодируется.

---

## Альбомный режим (ландшафт)

Типичная пара **ширина ≥ высота**, пресеты вроде **800×480** или **1280×720**.

```bash
./run.sh -r 1280x720 --video-scale stretch
# или по умолчанию 800×480:
./run.sh --video-scale stretch
```

Для тача в протоколе объявляются те же размеры, что и окно; выбор enum видео на стороне HU согласован с альбомными пресетами (см. `hu_aap.py`: режимы 1–5 в `VIDEO_RESOLUTION`).

---

## Портретный режим (книжный)

Цель — получить поток **720×1280** (или другой портретный пресет из enum **6–9**), без «перевёрнутого» ландшафтного потока на вертикальном экране.

Пример запуска:

```bash
./run.sh --video-debug -r 720x1280 --video-preset 720x1280 --touch-mode native --video-scale stretch
```

Кратко по флагам:

| Флаг | Зачем |
|------|--------|
| **`-r 720x1280`** | Окно и логические размеры под книжную ориентацию. |
| **`--video-preset 720x1280`** | Явный enum `VIDEO_RESOLUTION_720x1280` в `VideoConfig` (в протоколе AA нет произвольного WxH — только таблица пресетов + поля margin). |
| **`--touch-mode native`** | В **ServiceDiscovery** в телефон уходит **touch_screen** как **реальные** W×H окна (720×1280), а не разворот в альбом **max×min** для handshake. По умолчанию в приложении уже **native**; режим **`auto`** подменяет портретное окно на альбомные размеры тача — удобно для капризных телефонов, но часто даёт **не** портретный видеопоток. |
| **`--video-scale stretch`** | Кадр на весь экран (как у типичного ГУ); **`letterbox`** — с полями по пропорциям. |

### Что пришлось допилить для портрета (чего часто нет в «голых» форках headunit/OpenAuto)

1. **Согласованная тройка** «touch UI → enum видео → margins»: в GAL разрешение кадра задаётся только **enum** + **margin_width/height** относительно объявленного **touch_screen** (`hu_aap.py`). Нужны **портретные** значения enum (6–9) и выбор только из них при **h > w** (`_pick_video_mode_for_touch_ui`).

2. **`--touch-mode native`**: отдельная ветка **`_protocol_touch_dims`** в `main.py` — объявлять телефону портретный touch как есть. Во многих проектах оставляют только ландшафтный handshake (аналог **`auto`**), тогда телефон выбирает ландшафтный пресет, а картинка на вертикальном дисплее оказывается «не той» ориентации без лишних костылей в рендере.

3. **Явный `--video-preset`** для стабильного совпадения с ожидаемым кадром (особенно если автоподбор по тачу даёт другой ближайший пресет).

4. **Декодирование**: PyAV + **NumPy** для RGB/pygame; первая порция может быть AVCC — это нормально.

5. **DPI и положение руля**: см. ниже — **`--dpi-scale`**, **`--dpi`**, **`--driver-position`** (детали в `./run.sh --help` и в `source/main.py`).

## DPI (VideoConfig)

Поле **dpi** в ServiceDiscovery задаётся формулой по площади touch UI и **`--dpi-scale`**, либо явно флагом **`--dpi`** (см. `./run.sh --help` из каталога `source/`).

**Портрет 1080×1920** (`-r 1080x1920`, книжный экран): в этой конфигурации стабильная работа наблюдается при **dpi не выше 240**; больше 240 — типично перестаёт корректно работать. Подбирайте **`--dpi`** до 240 или снижайте **`--dpi-scale`**, чтобы итоговая формула не давала dpi > 240 для этого режима. Константа для ссылок в коде: **`VIDEO_DPI_PORTRAIT_1080X1920_MAX`** в `source/hu_aap.py`.

---

## Положение руля (ServiceDiscovery)

Флаг **`--driver-position lhd|rhd`** задаёт поле **`driver_pos`** в ответе ServiceDiscovery: **lhd** — левый руль (LHD, по умолчанию), **rhd** — правый руль (RHD). В protobuf для телефона: **`False` — LHD**, **`True` — RHD**. Телефон может подстраивать разметку под сторону водителя.

## TLS и сертификаты (Jaguar Land Rover OEM)

Протокол AA после handshake идёт по **TLS**; головное устройство выступает **TLS-клиентом** к телефону и предъявляет **клиентский** сертификат OEM (как в штатных решениях класса Headunit Reloaded / реверс HU).

В каталоге **`source/certs/`** лежат:

| Файл | Назначение |
|------|------------|
| **`jaguar.crt`**, **`jaguar.key`**, **`jaguar_chain.crt`** | Клиентская пара и цепочка для режима **`--cert jaguar`** (по умолчанию). Подпись в духе **Jaguar Land Rover OEM** (`hu_ssl.py`). |
| **`lr.crt`**, **`lr.key`**, **`lr_chain.crt`** | То же для **`--cert lr`** (Land Rover). |
| **`google_aa_ca.crt`** | CA для проверки цепочки со стороны телефона / инфраструктуры AA (подгрузка в контекст OpenSSL, см. `hu_ssl.py`). |
| **`hu.crt` / `hu.key`** | Лежат в `certs/` рядом с OEM-цепочками; текущий код TLS в **`hu_ssl.py`** для клиента использует только **jaguar** или **lr** (см. выше). |

Запуск с явным выбором (если не нужен jaguar по умолчанию):

```bash
./run.sh --cert jaguar …
./run.sh --cert lr …
```

Сертификаты взяты из практики портов **gartnera/headunit** и реверса **Android Auto / HU**; используйте только в рамках своего оборудования и приватного репозитория.
